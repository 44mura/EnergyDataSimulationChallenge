1. Decide an algorithm for this challenge

To decide a prime candidate for the algorithm solving this challenge, I go through the data-set and sort out the points of the results below:

  • The data-set is a time series.
  • However, it is only 23 months' worth of data.
  • Unfortunately the number of unique data in the data-set is just 28 in spite of data-set obtained from 500 houses, and it means that many houses have exactly the same time series data. I think that the model made from such a data-set cannot be evaluated accurately.

For the reason listed above, I first decide to delete overlapped data to make unique data-set (23 months' data-set of 28 houses). In this case, the prediction model for the future of the process based on its present or previous state such as time series analysis and markov process are suited for such a data-set. In the case of time series analysis, a decrease of data-set doesn't matter much because the prediction model is basically built with only each previous data per house.

I understand the data-set, which cover just 23 months, may not be enough, but as a prime candidate for the algorithm attacking this challenge, I decide to build a prediction model with time series analysis.

2. Format the data-set for Time Series Analysis

Through this challenge, "statsmodels" library is used for time series analysis. The docs is here http://statsmodels.sourceforge.net/

For time series analysis, it is very important that the time series data is stationary. To confirm the state, I plot "house 1" data as an example. Before plotting, I already confirmed each house has a similar wave shape.

From the above charts, EnergyProduction and Daylight have trend components and week seasonal components, and also Temperature has very strong seasonal component. It means the data-set is not stationary. To remove the trend components, the first differences(1-Diff) of the three variables are taken and plotted.

From the above charts, trend components of EnergyProduction and Daylight is mostly removed. However, seasonal components of three variables, especially Temperature, are not removed yet. I attacked this problem, but the seasonal components could not be removed meaningfully because the data-set cover only 23 months, even if the period of the data-set is 12 months.
Before making a model, I take a unit root test (Dickey-Fuller test) to confirm the trend components are really removed.

3. Build and evaluate a prediction model

In this case, the model is built with Vector autoregression (VAR) per house.

def mape_of_var(house, predfile, mapefile):
    sum0 = 0
    f_pred = open(predfile, 'w')
    f_mape = open(mapefile, 'w')
    pred_writer = csv.writer(f_pred)
    pred_writer.writerow(['House','EnergyProduction'])
    for i in house:
        data = data_train[data_train.House==i][['EnergyProduction','Daylight','Temperature']].set_index(rng)
        data_diff = data.diff().dropna()
        model = var.var_model.VAR(data_diff)
        results = model.fit(4, trend='nc')
        lag_order = results.k_ar
        data_pred = results.forecast(data_diff.values[-lag_order:],1)[0,0]+data.EnergyProduction[-1]
        sum0 = sum0 + abs(data_pred-data_test.EnergyProduction[i-1])/data_test.EnergyProduction[i-1]
        pred_writer.writerow([i,data_pred])

    mape = round(sum0/len(house),3)
    f_mape.write(str(mape))
    f_pred.close()
    f_mape.close()

    return mape
